# 📋 Фрагменти коду та функціоналу Kronos

У цій папці містяться виділені фрагменти інтерфейсу та логіки для наступних розділів симулятора торгівлі:
1. **Показники (Metrics)** — загальний капітал, чистий прибуток, середній вхід та нефіксований PnL.
2. **Аналітика останніх 5 угод (Trades Analytics)** — розрахунок середнього курсу купівлі/продажу, сумарна комісія та таблиця останніх транзакцій.
3. **Статистика курсу (останні 1500 оновлень) (Price History Statistics)** — мінімуми/максимуми, середнє значення, рівні 25% та 75%, налаштування вікна та бази тіків, розрахунок віртуальної прибутковості.
4. **Графік історії показників (Plotly Chart)** — побудова кривої поточного курсу, рівнів опору/підтримки та точок входу віртуальної стратегії.

## 📂 Структура файлів

### 1. 🖥️ [interface.html](file:///Users/kostantinkrivula/Desktop/Kronos-master/copy/interface.html)
* **Що містить:** HTML-розмітка для контейнера `.metrics-section` та вбудовані CSS стилі.
* **Стилі:** Включає змінні колірної схеми кавової теми (`--coffee-`), сітку карток `.exchange-wallet-grid`, клас `.exchange-card`, а також табличну структуру для статистики й порожній контейнер для графіка Plotly (`#metrics-history-chart`).

### 2. ⚡ [frontend_logic.js](file:///Users/kostantinkrivula/Desktop/Kronos-master/copy/frontend_logic.js)
* **Що містить:** Клієнтська логіка на JavaScript.
* **Основні функції:**
  * `fetchBtcRealTimePrice()` та `startBtcPricePolling()` — опитування API Binance кожні 5 секунд для отримання актуальної вартості BTC.
  * `addBtcPriceToHistory(price)` — додавання нових значень та зміщення (shift) застарілих елементів відповідно до налаштованого розміру вікна (`btcHistoryWindowSize`).
  * `recalculateAllHistoryMetrics()` та `handleWindowSizeChange(value)` — перерахунок масивів показників при зміні кількості відображуваних тіків.
  * `updatePriceHistoryMetrics()` — обрахунок статичних показників (середнє, мін/макс, рівні 25% та 75%) та **автоматичний запуск віртуальної стратегії** торгівлі за рівнями:
    * *Купівля:* курс $\le$ 25% рівень.
    * *Продаж:* курс $\ge$ 75% рівень (з урахуванням покриття комісії та виходу в беззбитковість).
  * `updatePriceHistoryChart()` — рендеринг графіку за допомогою бібліотеки `Plotly.js` (згладжування сплайнів, заливка областей, нанесення спеціальних маркерів точок входу).
  * `updateTradesAnalytics()` — завантаження історії угод з бекенду, розрахунок агрегованої аналітики та побудова детальної таблиці останніх 5 транзакцій.
  * `updateExchangeUI()` — оновлення стану гаманця користувача.

### 3. 🐍 [backend_logic.py](file:///Users/kostantinkrivula/Desktop/Kronos-master/copy/backend_logic.py)
* **Що містить:** Серверна логіка на Python (Flask + потоки).
* **Основні компоненти:**
  * `tick_accumulation_loop()` — фоновий демон-потік, який збирає тикові дані з Binance API та наповнює глобальний масив `ticks_list` в оперативній пам'яті.
  * `save_ticks_history()` / `load_ticks_history()` / `save_ticks_to_supabase()` — збереження та завантаження тиків у локальний файл `ticks_history.json` та синхронізація з Supabase Cloud.
  * `register_shutdown_handlers()` — обробники системних сигналів (SIGTERM, SIGINT) для гарантованого збереження буфера пам'яті на диск при перезавантаженні сервера.
  * Flask маршрути:
    * `/api/exchange/ticks-history` — віддача історії тиків для ініціалізації фронтенду.
    * `/api/exchange/tick-database-size` — зміна максимального обсягу бази тіків на сервері (до 100 000).
    * `/api/exchange/history` — отримання списку проведених реальних угод користувача для аналітики.

## 🛠️ Інтеграція
Щоб використати цей функціонал у власному проекті:
1. Перенесіть HTML та стилі з `interface.html` у відповідний шаблон.
2. Підключіть скрипт `frontend_logic.js` (переконайтеся, що на сторінці підключені бібліотеки `Plotly.js` та `Axios`).
3. Додайте Python-код з `backend_logic.py` у свій Flask-додаток та запустіть фоновий потік `tick_accumulation_loop` під час старту веб-сервера.
